# PSD 布局标签使用指南 (Layout Tags Guide)

## 简介
为了让生成的 HTML/CSS 代码更加符合现代 Web 响应式前端布局的标准，我们的解析引擎支持在 PSD 的**图层或图层组名称**中添加特定的**布局标签 (Layout Tags)**。
利用这些标签，你可以通过命名直接控制节点在页面中的流式布局（Flex）或特殊定位方式（Fixed / Absolute），打破传统的绝对坐标（Absolute Positioning）硬编码，生成高可维护性的代码。

## 支持的布局标签

无论是单独的图层还是图层组，只要在其名称的**最开头**加上对应的 `[tag]` 即可生效。例如：`[flow-y]导航栏`、`[fixed]悬浮按钮` 等。

### 1. `[flow-y]` (垂直流式布局)
- **作用**：将当前图层组转换为 Flex 垂直容器 (`display: flex; flex-direction: column`)。其内部的子元素会按 `y` 坐标从上到下进行垂直排序。
- **排版计算**：系统会自动计算垂直方向上相邻子元素之间的间距，并转换为 Web 标准的 `margin-top`。多级嵌套可实现完美的上下流式堆叠，真实还原设计稿留白。
- **适用场景**：长列表、垂直排版的详情页、由多个上下堆叠模块组成的页面主容器。
- **示例**：`[flow-y] main-content`

### 2. `[flow-x]` (水平流式布局)
- **作用**：将当前图层组转换为 Flex 水平容器 (`display: flex; flex-direction: row`)。其内部的子元素会按 `x` 坐标从左到右进行水平排序。
- **排版计算**：系统会自动剥离相邻子元素之间的水平间距，转换为 `margin-left` 属性，保留原生的紧凑度。如果有局部的纵向偏移，也会作为辅轴边距被计算在内。
- **适用场景**：水平的一排图标菜单、横向靠右排列的按钮组、标签栏 (Tabs) 等。
- **示例**：`[flow-x] tab-buttons`

### 3. `[fixed]` (固定定位)
- **作用**：该图层/图层组将采用固定定位 (`position: fixed`)，具备高视图层级 (`z-index: 100`)，脱离常规文档流。在页面滚动时，它始终悬浮在浏览器的固定位置。
- **响应式计算**：
  - **水平居中响应**：为防止固定元素在宽屏下偏离内容区，系统不会死板地绑定左边距（`left`），而是以屏幕水平中心为基准 (`left: 50%`)，加上原始坐标对于设计稿中线的偏移 (`margin-left`)。这使得悬浮物能够永远相对页面主体“粘连式”展示。
  - **垂直智能吸附**：如果图层位于 PSD 第一屏内（例如 `Y坐标 < 900px`），它会保持原生的顶端距离；若出现于更靠下的位置，系统将推断设计师意图为“垂直居中悬浮”，自动应用 `top: 50%; transform: translateY(-50%)`。
- **适用场景**：侧边客服悬浮层、右下角的返回顶部按钮、全局的底层背景装饰或吸顶导航条。
- **示例**：`[fixed] float-server`

### 4. `[abs]` (绝对定位 - 免除流式干扰)
- **作用**：强制使用绝对定位 (`position: absolute`)。**当它身处于 `[flow-x]` 或 `[flow-y]` 这样的流式布局容器中时，该标签非常重要**。带有 `[abs]` 的元素会被流式计算“豁免”，不参与周围元素的 Flex 排序，而是静默地维持自己相对父级的原始绝对坐标 (`left` 和 `top`)。
- **适用场景**：大背景上的局部光效、卡片右上角悬停的“NEW”角标、叠加在流式内容上的印章特效等。
- **示例**：`[abs] new-badge`

## 规则与注意事项
1. **组合嵌套**：你可以随心所欲地嵌套标签。例如，在一个被设定为 `[flow-y]` 的页面外框下，包含数个 `[flow-x]` 的水平列表项。
2. **标签位置**：必须放置于图层或图层组名称的**最开头**，且被纯英文半角的方括号 `[]` 紧密包裹。例如 `[flow-y] list`。不能写成 `list [flow-y]` 或者 `【flow-y】`。
3. **命名规范**：标签后的图层原名尽量保证**语义化**与**合法性**（最好是英文短横线风格，如 `header-bar`）。解析器最终会将它们编译为相应的 CSS 类名。
